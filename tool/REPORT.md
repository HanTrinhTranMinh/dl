# RT-DETR Code Audit & Technical Report

## 0) Executive Summary

### High-level Architecture Overview
RT-DETR (Real-Time Detection Transformer), as implemented in this repository, is an end-to-end object detector designed to eliminate the need for Non-Maximum Suppression (NMS) during inference while achieving real-time speeds. The architecture consists of three main components:
1. **Backbone**: A CNN (default PResNet) extracting multi-scale features.
2. **Hybrid Encoder**: A cross-scale feature fusion module that utilizes a lightweight attention layer (Intra-scale) and a CNN-based top-down/bottom-up PANet (Cross-scale).
3. **Transformer Decoder**: A multi-layer decoder employing Multi-Scale Deformable Attention, processing fixed queries into bounding box predictions and class probabilities iteratively.

### Key Differences from Original DETR
- **Hybrid Encoder**: Replaces the standard dense self-attention encoder with a hybrid approach (CNN-based PANet + lightweight single-scale Transformer).
- **Efficient Deformable Attention**: Uses sparse sampling points (default 4) per reference point across multiple levels instead of dense global attention.
- **IoU-aware Query Selection**: Queries are initialized not randomly, but from the top-k encoder features based on predicted objectness scores (`enc_score_head`).
- **No NMS**: Inference directly selects the top 300 queries based on focal loss thresholds (sigmoid logic) via `RTDETRPostProcessor`, bypassing NMS logic entirely.
- **Contrastive Denoising (CDN)**: Incorporates GT boxed with noise during training to stabilize bipartite matching and accelerate convergence.

### Top 10 Critical Things to Understand When Debugging RT-DETR
1. **No NMS**: Do not apply NMS. The `RTDETRPostProcessor` directly outputs the top-k predictions based on raw probabilities.
2. **Target Box Format**: Ground-truth target boxes fed into the model MUST be in normalized `(cx, cy, w, h)` format [0, 1].
3. **Losses are Dict-based**: The `rtdetr_criterion` returns a dict of losses spanning all auxiliary decoder layers AND denoising layers.
4. **Matcher Input Logits**: The Hungarian Matcher uses focal-loss style approximations. Logits are unactivated; focal scaling is applied internally using Sigmoid.
5. **Denoising Sub-split**: Outputs are split into denoising (`dn_aux_outputs`) and matching predictions dynamically. A shape mismatch here indicates wrong target formats.
6. **Query Initialization**: Initial reference points are generated by the `enc_bbox_head` (Encoder head predictions), then passed to the decoder.
7. **Positional Embeddings in Decoder**: Reference points are mapped through a 2-layer MLP (`query_pos_head`) to form query positional embeddings for cross-attention.
8. **Multi-Scale Feature Strides**: Features must perfectly align with predicted strides (8, 16, 32). Incorrect spatial shapes crash the MS Deformable Attn.
9. **Sigmoid + Inverse Sigmoid Trick**: Bounding box coordinates iteratively refine relative to `inverse_sigmoid(RefPoints)`. They are always constrained to (0, 1) via final `.sigmoid()`.
10. **Varifocal Loss (VFL)**: Classification is heavily weighted by IoU. A zero IoU early on collapses VFL, pulling class scores to 0 rapidly.

## 2) End-to-End Data Flow

### Training Pipeline
1. **Dataloader**: Converts boxes to `cxcywh`, normalizes by image dimensions, resizes images to `(640, 640)`.
2. **Backbone**: Computes multi-scale features (C3, C4, C5).
3. **Hybrid Encoder**: Fuses cross-scale context, applies intra-scale attention on C5, outputs [H/8, H/16, H/32] multi-scale features.
4. **Query Init**: Encoder head predictions act as objectness scores to select top 300 anchors.
5. **Denoising**: Ground truth targets with noise injected are concatenated with the queries.
6. **Decoder**: Queries pass through 6 alternating self-attention and MSDeformAttention layers, iteratively refining their anchor boxes.
7. **Loss/Criterion**: Predictions separated into denoising vs matching groups. `HungarianMatcher` finds optimal 1-to-1 assignments.

### Inference Pipeline
Same up to step 6, but **Denoising is completely skipped**.
7. **PostProcessor**: Takes final decoder `pred_logits` and `pred_boxes`. Converts to `xyxy`, scales by original image size, sigmoid on logits to find top 300 probabilities, and outputs final detections. No NMS.

### 3.1 Notation
- **B**: Batch Size
- **C**: Channels
- **H, W**: Image Height and Width (e.g., 640x640)
- **N**: Sequence Length (Usually $H \times W$ flattened or number of queries)
- **L**: Multi-Scale Levels (Level 0: Stride 8; Level 1: Stride 16; Level 2: Stride 32)
- **Q**: Number of queries (Default: 300)

### 3.3 Flattening & Reshaping
- **Encoder Self-Attention**: `P5` features `(B, 256, H/32, W/32)` are flattened starting from layer 2 via `.flatten(2).permute(0, 2, 1)`, resulting in `(B, (H/32)*(W/32), 256)`.
- **Decoder Cross-Attention**: Multi-Scale Deformable Attention takes flattened sequence from all scales concatenated together: `(B, 8400, 256)`. It uses `level_start_index = [0, 6400, 8000]` to unroll relationships conceptually without unflattening.
- **Positional Encodings**: `build_2d_sincos_position_embedding` builds 2D combinations of sin/cos. In Decoder, reference points undergo `inverse_sigmoid` and act as structural embedding. The shape `(B, Q, 4)` goes through a 2-layer MLP `query_pos_head` to form positional embeddings `(B, Q, 256)`.

### 3.4 Query Pipeline
- **Query Initialization (`learnt_init_query=False` in configs)**: Queries are strictly initialized dynamically using the Hybrid Encoder features. The model maps features through `enc_output`, passes them to `enc_score_head` and `enc_bbox_head`. Top-k (default 300) anchors based on the max classification value act as primary `reference_points`, extracting matching feature slices as `target`.
- **Target Extraction**: Top 300 `output_memory` elements form the query target embeddings `(B, 300, 256)`.

## 4) Deep Dive Per Module

### 4.1 Backbone
- **Implementation**: `src/nn/backbone/presnet.py` (PResNet variant D).
- **Core characteristic**: `return_idx=[1, 2, 3]`. Pulls stages mapping to C3, C4, C5 (strides 8, 16, 32).
- Channels: ResNet50 yields `[512, 1024, 2048]`.

### 4.2 Hybrid Encoder
- **Implementation**: `src/zoo/rtdetr/hybrid_encoder.py`.
- **Intra-scale Interaction**: Before cross-scale matching, C5 `(2048 -> 256)` goes through a generic 1-layer MultiheadAttention `TransformerEncoder`.
- **Cross-scale Fusion**: Employs CCSP-based blocks (`CSPRepLayer` containing `RepVggBlock`) in a standard FPN/PAN arrangement.
- **Output**: 3 resolution levels of identical dimension (256).

### 4.3 Transformer Decoder
- **Structure**: 6 decoder layers sequentially refining reference queries.
- **Self-Attention**: Standard multi-head attention `(Q, K, V are all target queries)`.
- **Cross-Attention**: `MSDeformableAttention` module explicitly querying 4 points per feature level from the dense flattened memory.
- **Iterative Refinement**: `inter_ref_bbox = F.sigmoid(bbox_head[i](output) + inverse_sigmoid(ref_points_detach))`. The boxes are iteratively offset layer by layer maintaining strict `(0, 1)` bounds. Format is invariably `cxcywh`.

### 4.4 Detection Head
- **Mechanism**: Dedicated Multi-Layer Perceptrons connected at EVERY decoder layer.
- **Logits**: Single linear layer mapping `hidden_dim -> num_classes`. NOT followed by softmax in network (raw logits are manipulated externally for focal loss).
- **Boxes**: 3-layer MLP predicting delta center, delta width, delta height mapping `hidden_dim -> 4`.

### 4.5 Losses
- **Criterion**: `rtdetr_criterion.py:SetCriterion`.
- **Classification (`loss_vfl`)**: Varifocal Loss. Blends IoU scores into the binary classification map to assign soft labels. Pulls focus on well-localized predictions.
- **Bbox Reg (`loss_bbox` and `loss_giou`)**: L1 distance across `(cx, cy, w, h)`, and Generalized Intersection over Union calculated after conversion to `xyxy`.
- **Weighting**: VFL (1.0), BBox (5.0), GIoU (2.0). Applied equivalently to all decoder steps (auxiliary).

### 4.6 Hungarian Matcher
- **Implementation**: `rtdetr_pytorch/src/zoo/rtdetr/matcher.py`.
- **Matrix Cost**: Computed linearly across all proposals -> `Self.cost_bbox * L1 + Self.cost_class * FOCAL + Self.cost_giou * GIOU`.
- **Formatting**: Internally flattens logits `out_prob = F.sigmoid(...)` and applies `torch.cdist` for pairs. Uses `scipy.optimize.linear_sum_assignment` for minimal bipartite cost.

### 4.7 Postprocessing
- **NMS**: **Completely Absent**.
- **Mechanics**: `rtdetr_postprocessor.py:RTDETRPostProcessor`. Uses Sigmoid to convert logits to probabilities, then sorts via `torch.topk(..., 300)`.
- Re-scales bounded `cxcywh` output using original image size tensor. Detections returned identically to COCO expected properties.

## 5) Config & Hyperparameters
*(Gathered from `rtdetr_r50vd_6x_coco.yml` and includes)*
- **Classes**: `80` (remap mapping inside configs enables 91 conversion for standard COCO logic).
- **Queries (`num_queries`)**: `300`. Expected to retrieve at maximum 300 valid boxes per picture.
- **Denoising (`num_denoising`)**: `100`. Accelerates convergence dynamically appending noise boxes.
- **Image Size**: Dynamic batch scaling from `[480, ... 800]` but strict `[640, 640]` evaluation size.
- **Target Transform**: `SanitizeBoundingBox`, `ConvertBox(out_fmt='cxcywh', normalize=True)`.
- **Optimizer**: AdamW. LR = `0.0001` with Backbone LR reduced by factor 10 (`0.00001`). Weight Decay: `0.0001`.
- **EMA setup**: ModelEMA decay rate `0.9999` initialized at 2000 steps warmup.

## 6) Common Failure Points Causing mAP/Recall = 0

| Symptom | Root Cause | Code Location | How to Verify | How to Fix |
|---|---|---|---|---|
| mAP=0 early epochs | **Wrong Target BBox Format**. Model inherently assumes GTs are `(cx,cy,w,h)` normalized. | `rtdetr_criterion.py:SetCriterion:loss_boxes()` | Check `targets["boxes"]`. Values must be bounding box fraction `(0.1 - 0.9)`. | Pass normalized `cxcywh` natively, or verify transform `ConvertBox` presence. |
| Gradients Exploding in Head | **Lack of inverse_sigmoid limits**. Raw targets outside (0,1) break inv_sigmoid. | `rtdetr_decoder.py:TransformerDecoder` | Check `ref_points_detach`; if logit explodes, targets were likely non-normalized. | Clamp targets inside dataset loader: `SanitizeBoundingBox(min_size:1)` |
| Recall=0 forever | **VFL Zero Collapse**. Network assigns zero IoU continuously, causing VFL multiplier to stay 0. | `loss_labels_vfl` | Dump `ious = torch.diag(ious)` inside VFL loss; if completely 0, gradient dies. | Change to CrossEntropy debug-wise or fix matcher format. |
| Everything predicts class 0 | **Softmax Instead of Sigmoid used in match/loss**. | `matcher.py:HungarianMatcher` | Confirm `use_focal_loss=True` parameter passed during inference init | RT-DETR demands `F.sigmoid(logits)`. |
| Loss diverges > 100| **Aux logits mismatch targets dimension length**. | `SetCriterion.forward()` | Ensure `targets` dict is identical shape/len as `dn_aux_outputs`. | Do not prune targets selectively for one layer. |
| Denoising crashes shape errors| **`num_classes` embedding dimension mismatch**. | `RTDETRTransformer.__init__` | Compare `self.denoising_class_embed` which inherently assumes `+1` padding. | Ensure embedding pad config matches vocabulary limits exactly. |
| Recall drops at Eval | **Missing EMA parameter restoration**. Eval runs on un-smoothed weights. | `solver.py` / `val_dataloader` | Compare weights before eval hook vs EMA. | Call `ema.apply_shadow()` before launching eval functions. |
| mAP stops around 10% | **Positional Encodings undefined dynamically**. | `HybridEncoder:forward` | Verify `pos_embed` presence during evaluation scale dynamic. | Use strictly multiple of `32` input matrices size at inference. |
| 0 Detections visualized | **Postprocessor not multiplying original sizes**. Returns boxes spanning (0, 1) pixel plane. | `rtdetr_postprocessor.py` | Visualize numerical outputs post-processor. They should be `cx * w_orig`. | Verify dataset `orig_target_sizes` mapping propagates. |
| Matcher Error - Shape | **Matcher receives dict incorrectly extracted**. Denoising queries mixed up with real predictions. | `RTDETRCriterion` | Look sequentially inside matching iteration for `outputs_without_aux`. | Do not route `dn_aux` to `matcher.py` natively. |
| Zero mAP | **Matcher cost weights set to 0** | `rtdetr_r50vd.yml:SetCriterion:matcher` | Verify `cost_class: 2`, `cost_bbox: 5`. | Restore baseline configs. |
| Box Loss stays ~1.0| **Using pixel scale target**. L1 compares 0.5 with 640. | `SetCriterion:loss_boxes` | Check `src_boxes` max vs `target_boxes` max. | Normalize targets by dataset `img_shape`. |
| Class error ~99% | **Focal loss alpha param inverted or unset.** | `SetCriterion:loss_labels_focal` | `alpha 0.75, gamma 2.0` | Reset focal params to RTDETR baseline. |
| NaN Loss output | **Empty Targets in batch yielding div/0**. Target size 0 without `clamp(min=1)`. | `SetCriterion.forward()` | Track `num_boxes = sum(len(t))` closely. | Repository code clamps `num_boxes / get_world_size()` cleanly. |
| No convergence / recall | **Number of classes in network vs config mismatch**. `num_classes=80` hardcoded. | `dataset/coco_detection.yml` | Check output dimension of `enc_score_head`. | Ensure `num_classes` propagated across Criterion, Matcher, and Head identically. |

## 7) How to Instrument Shapes

In debug workflows, isolating the `Hybrid Encoder` output and `Decoder` iterations is necessary. A custom script hooking into `forward` allows extraction without polluting the repository files.

*(A fully running shape instrumentation script has been delivered. See `dump_shapes.py` in the directory root.)*

Code Snippet Summary for Forward Hooks:
```python
def print_shape_hook(module, input, output):
    print(f"--- Output of {module.__class__.__name__} ---")
    if isinstance(output, torch.Tensor):
         print(output.shape)
    elif isinstance(output, list) and isinstance(output[0], torch.Tensor):
         print(f"List of tensors, length {len(output)}, shape[0]: {output[0].shape}")

# Best insertion points:
model.encoder.register_forward_hook(print_shape_hook)
model.decoder.decoder.register_forward_hook(print_shape_hook) 

model(dummy_image)
```
